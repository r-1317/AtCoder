A - Molecules /

実行時間制限: 2 sec / メモリ制限: 1024 MiB

ストーリー

AtCoder研究所では、新しい分子の開発を行っている。 天才化学者である髙橋くんは、動き回る原子同士を好きなタイミングで結合させて、分子を作ることができる。 ただし、原子を結合するには距離に応じてエネルギーを消費してしまうので、できるだけ少ないエネルギーで決められた数の分子を作りたい。

問題文

0≤x<L=1050 \leq x < L = 10^50≤x<L=105、0≤y<L=1050 \leq y < L = 10^50≤y<L=105 からなる2次元平面を考える。ただし、この平面は左右端（x=0x = 0x=0 と x=Lx = Lx=L）、および上下端（y=0y = 0y=0 と y=Ly = Ly=L）がそれぞれつながっており、全体としてトーラス構造となっている。範囲外の座標は、xxx および yyy をそれぞれ LLL で割った余りに置き換えることで、常に 0≤x,y<L0 \leq x, y < L0≤x,y<L の範囲に正規化される。例えば、位置 (10000,90000)(10000, 90000)(10000,90000) から (−20000,30000)(-20000, 30000)(−20000,30000) 進んだ先の座標は (90000,20000)(90000, 20000)(90000,20000) である。

この平面上に NNN 個の点が存在する。t=0t = 0t=0 において、iii 番目の点（0≤i<N0 \leq i < N0≤i<N）の初期位置 (xi,yi)(x_i, y_i)(xi​,yi​)（0≤xi,yi<L0 \leq x_i, y_i < L0≤xi​,yi​<L）と初期速度 (vxi,vyi)(vx_i, vy_i)(vxi​,vyi​)（−100≤vxi,vyi≤100-100 \leq vx_i, vy_i \leq 100−100≤vxi​,vyi​≤100）が入力で与えられる。初期状態では、各点は独立した連結成分を形成している。

各時刻 t=0,1,…,T−1t = 0, 1, \ldots, T - 1t=0,1,…,T−1 において、以下の 1. 結合フェーズ と 2. 移動フェーズ の順で交互に処理を行う。

1. 結合フェーズ

時刻 ttt において、異なる連結成分に属する 2 つの点を 結合 することができる。同一時刻に複数の結合を行ってもよい。

点 iii の時刻 ttt における位置を (xi,yi)(x_i, y_i)(xi​,yi​)、点 jjj の時刻 ttt における位置を (xj,yj)(x_j, y_j)(xj​,yj​) としたとき、点 iii と点 jjj を結合する際の 結合コスト DDD は、以下の距離の式で計算される：

D=round((min⁡(L−Δx, Δx))2+(min⁡(L−Δy, Δy))2) D = \mathrm{round}\left( \sqrt{ \bigl(\min(L - \Delta x,\ \Delta x)\bigr)^2 + \bigl(\min(L - \Delta y,\ \Delta y)\bigr)^2 } \right) D=round((min(L−Δx, Δx))2+(min(L−Δy, Δy))2​)

Δx=∣xi−xj∣,Δy=∣yi−yj∣ \Delta x = |x_i - x_j|,\quad \Delta y = |y_i - y_j| Δx=∣xi​−xj​∣,Δy=∣yi​−yj​∣

点を結合すると、運動量保存則に従って連結成分の速度が更新される。結合前の時点において、点 iii が速度 (vxA,vyA)(vx_A, vy_A)(vxA​,vyA​) で動く連結成分 AAA に属し、点 jjj が速度 (vxB,vyB)(vx_B, vy_B)(vxB​,vyB​) で動く連結成分 BBB に属しているとする。連結成分 AAA および連結成分 BBB に属しているすべての点について、結合後の連結成分の速度 (vxnew,vynew)(vx_{\text{new}}, vy_{\text{new}})(vxnew​,vynew​) は以下のように更新される：

(vxnew,vynew)=(∣A∣×vxA+∣B∣×vxB∣A∣+∣B∣,∣A∣×vyA+∣B∣×vyB∣A∣+∣B∣) (vx_{\text{new}}, vy_{\text{new}}) = \left( \frac{|A| \times vx_A + |B| \times vx_B}{|A| + |B|}, \frac{|A| \times vy_A + |B| \times vy_B}{|A| + |B|} \right) (vxnew​,vynew​)=(∣A∣+∣B∣∣A∣×vxA​+∣B∣×vxB​​,∣A∣+∣B∣∣A∣×vyA​+∣B∣×vyB​​)

ここで、∣A∣|A|∣A∣、∣B∣|B|∣B∣ はそれぞれ連結成分に含まれる点の数を示す。結合後、その連結成分に属するすべての点は、同じ速度で動くようになる。結合によって点の位置は変化しない。また、同一時刻内での結合の順序は、結合後の位置、結合コスト、および連結成分の速度に影響しない。

座標および速度は小数となることがあるが、計算はすべて倍精度浮動小数（double）で管理される。

2. 移動フェーズ

各連結成分に属するすべての点の位置を同時に更新する。点 iii の時刻 ttt における位置を (xi,yi)(x_i, y_i)(xi​,yi​)、点 iii が属する連結成分の速度を (vxi,vyi)(vx_i, vy_i)(vxi​,vyi​) とすると、時刻 t+1t + 1t+1 における点 iii の位置 (xi′,yi′)(x_i', y_i')(xi′​,yi′​) は以下のようになる：

(xi′,yi′)=((xi+vxi) mod L,(yi+vyi) mod L) (x_i', y_i') = ((x_i + vx_i) \bmod L, (y_i + vy_i) \bmod L) (xi′​,yi′​)=((xi​+vxi​)modL,(yi​+vyi​)modL)

時刻 TTT において、連結成分数が ちょうど MMM 個、かつ各連結成分のサイズが ちょうど KKK となるように結合を計画し、結合コスト DDD の合計を最小化せよ。

結合と移動の例

上図では、3 つの点を 1 つの連結成分にまとめる例を示している。 まず最初に、左下と右下の 2 点を結合し、進行方向が上向きに変化する。 次に、上部を右向きに進む点と結合し、進行方向は右上に変化する。

得点

すべての結合コスト DDD の合計を DsumD_{\text{sum}}Dsum​ とする。このとき、1 テストケースの得点は以下のように計算される。得点は大きいほど良い。

Score=round ⁣(106×log⁡2 ⁣(L×(N−M)Dsum+1)) \mathrm{Score} = \mathrm{round}\!\left(10^{6} \times \log_{2}\!\left( \frac{L \times (N - M)}{D_{\text{sum}} + 1} \right)\right) Score=round(106×log2​(Dsum​+1L×(N−M)​))

以下の場合、WA となる。

  • 無効な出力を行った場合
  • 時刻 TTT 時点で連結成分数が ちょうど MMM 個、各連結成分のサイズが ちょうど KKK でない場合
  • すでに同一連結成分に属している点の間で結合が指定された場合

合計で 150 個のテストケースがあり、各テストケースの得点の合計が提出の得点となる。 一つ以上のテストケースで不正な出力や制限時間超過をした場合、提出全体の判定がWAやTLEとなる。 コンテスト時間中に得た最高得点で最終順位が決定され、コンテスト終了後のシステムテストは行われない。 同じ得点を複数の参加者が得た場合、提出時刻に関わらず同じ順位となる。

Story

AtCoder Laboratory is engaged in the development of new molecules. Takahashi, a genius chemist, can freely bond atoms that move around at any timing he chooses to form molecules. However, bonding atoms consumes energy depending on the distance between them, so he wants to create the required number of molecules using as little energy as possible.

Problem Statement

Consider a two-dimensional plane defined by 0≤x<L=1050 \leq x < L = 10^50≤x<L=105 and 0≤y<L=1050 \leq y < L = 10^50≤y<L=105. This plane has a toroidal structure, meaning the left and right edges (x=0x = 0x=0 and x=Lx = Lx=L), as well as the top and bottom edges (y=0y = 0y=0 and y=Ly = Ly=L), are connected. Any coordinates outside this range are normalized to fall within 0≤x,y<L0 \leq x, y < L0≤x,y<L by taking the remainder when divided by LLL for both xxx and yyy. For example, the position reached by moving (−20000,30000)(-20000, 30000)(−20000,30000) from (10000,90000)(10000, 90000)(10000,90000) is (90000,20000)(90000, 20000)(90000,20000).

There are NNN points on this plane. At time t=0t = 0t=0, the initial position (xi,yi)(x_i, y_i)(xi​,yi​) (0≤xi,yi<L0 \leq x_i, y_i < L0≤xi​,yi​<L) and initial velocity (vxi,vyi)(vx_i, vy_i)(vxi​,vyi​) (−100≤vxi,vyi≤100-100 \leq vx_i, vy_i \leq 100−100≤vxi​,vyi​≤100) of the iii-th point (0≤i<N0 \leq i < N0≤i<N) are given as input. In the initial state, each point forms an independent connected component.

At each time t=0,1,…,T−1t = 0, 1, \ldots, T - 1t=0,1,…,T−1, the following two phases are processed in order: 1. Bonding Phase and 2. Movement Phase.

1. Bonding Phase

At time ttt, it is possible to bond two points that belong to different connected components. Multiple bonds may be performed in the same time step.

Let (xi,yi)(x_i, y_i)(xi​,yi​) be the position of point iii at time ttt, and (xj,yj)(x_j, y_j)(xj​,yj​) be the position of point jjj at time ttt. The bonding cost DDD for bonding point iii and point jjj is calculated using the following distance formula:

D=round((min⁡(L−Δx, Δx))2+(min⁡(L−Δy, Δy))2) D = \mathrm{round}\left( \sqrt{ \bigl(\min(L - \Delta x,\ \Delta x)\bigr)^2 + \bigl(\min(L - \Delta y,\ \Delta y)\bigr)^2 } \right) D=round((min(L−Δx, Δx))2+(min(L−Δy, Δy))2​)

Δx=∣xi−xj∣,Δy=∣yi−yj∣ \Delta x = |x_i - x_j|,\quad \Delta y = |y_i - y_j| Δx=∣xi​−xj​∣,Δy=∣yi​−yj​∣

When two points are bonded, the velocity of the resulting connected component is updated according to the law of conservation of momentum. Suppose that before the bond, point iii belongs to connected component AAA moving at velocity (vxA,vyA)(vx_A, vy_A)(vxA​,vyA​), and point jjj belongs to connected component BBB moving at velocity (vxB,vyB)(vx_B, vy_B)(vxB​,vyB​). Then, for all points belonging to connected components AAA and BBB, the velocity (vxnew,vynew)(vx_{\text{new}}, vy_{\text{new}})(vxnew​,vynew​) after bonding is updated as follows:

(vxnew,vynew)=(∣A∣×vxA+∣B∣×vxB∣A∣+∣B∣,∣A∣×vyA+∣B∣×vyB∣A∣+∣B∣) (vx_{\text{new}}, vy_{\text{new}}) = \left( \frac{|A| \times vx_A + |B| \times vx_B}{|A| + |B|}, \frac{|A| \times vy_A + |B| \times vy_B}{|A| + |B|} \right) (vxnew​,vynew​)=(∣A∣+∣B∣∣A∣×vxA​+∣B∣×vxB​​,∣A∣+∣B∣∣A∣×vyA​+∣B∣×vyB​​)

Here, ∣A∣|A|∣A∣ and ∣B∣|B|∣B∣ denote the number of points in connected components AAA and BBB, respectively. After bonding, all points in the resulting connected component will move with the same velocity. The positions of the points do not change due to bonding. Additionally, the order in which bonds are performed within the same time step does not affect the resulting positions, bonding costs, or velocities of the connected components.

Coordinates and velocities may become fractional, but all computations are performed using double-precision floating-point arithmetic.

2. Movement Phase

The positions of all points in each connected component are updated simultaneously. Let (xi,yi)(x_i, y_i)(xi​,yi​) be the position of point iii at time ttt, and let (vxi,vyi)(vx_i, vy_i)(vxi​,vyi​) be the velocity of the connected component to which point iii belongs. Then, the position (xi′,yi′)(x_i', y_i')(xi′​,yi′​) of point iii at time t+1t + 1t+1 is updated as follows:

(xi′,yi′)=((xi+vxi) mod L,(yi+vyi) mod L) (x_i', y_i') = ((x_i + vx_i) \bmod L, (y_i + vy_i) \bmod L) (xi′​,yi′​)=((xi​+vxi​)modL,(yi​+vyi​)modL)

Plan the bonds so that at time TTT, the number of connected components is exactly MMM, and the size of each connected component is exactly KKK, while minimizing the total bonding cost DDD.

Scoring

Let the total bonding cost of all bonds be DsumD_{\text{sum}}Dsum​. The score for a single test case is calculated as follows. A higher score is better.

Score=round ⁣(106×log⁡2 ⁣(L×(N−M)Dsum+1)) \mathrm{Score} = \mathrm{round}\!\left(10^{6} \times \log_{2}\!\left( \frac{L \times (N - M)}{D_{\text{sum}} + 1} \right)\right) Score=round(106×log2​(Dsum​+1L×(N−M)​))

The result will be WA in the following cases:

  • If the output is invalid
  • If at time TTT, the number of connected components is not exactly MMM, or the size of each connected component is not exactly KKK
  • If a bond is specified between two points that already belong to the same connected component

There are 150150150 test cases, and the score of a submission is the total score for each test case. If your submission produces an illegal output or exceeds the time limit for some test cases, the submission itself will be judged as WA or TLE , and the score of the submission will be zero. The highest score obtained during the contest will determine the final ranking, and there will be no system test after the contest. If more than one participant gets the same score, they will be ranked in the same place regardless of the submission time.
